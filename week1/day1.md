## 📖 이론 내용

### 1. 클라우드 컴퓨팅이란?

클라우드 컴퓨팅은 인터넷을 통해 IT 리소스와 애플리케이션을 온디맨드로 제공하는 서비스입니다. 물리적 서버를 직접 구매하고 유지 관리하는 대신, AWS와 같은 클라우드 공급자로부터 필요한 만큼만 컴퓨팅 파워, 스토리지, 데이터베이스 등을 사용하고 비용을 지불합니다.

**클라우드 컴퓨팅의 6가지 장점:**
1. **초기 비용 절감**: 고정 비용 대신 변동 비용 지불
2. **규모의 경제 실현**: 수많은 고객으로 인한 낮은 가격
3. **용량 추측 불필요**: 필요한 만큼 즉시 확장/축소
4. **속도 및 민첩성 향상**: 몇 분 만에 새 리소스 확보
5. **데이터 센터 운영 비용 절감**: 인프라 관리에 집중하지 않아도 됨
6. **글로벌 확장**: 몇 분 만에 전 세계 배포 가능

### 2. AWS 글로벌 인프라

AWS는 전 세계에 걸쳐 물리적 인프라를 운영합니다.

#### 리전 (Region)
- 지리적으로 분리된 데이터 센터의 클러스터
- 각 리전은 완전히 독립적
- 현재 30개+ 리전 운영 중
- 예: ap-northeast-2 (서울), us-east-1 (버지니아 북부)

**리전 선택 기준 (⭐ 시험 출제):**
1. **규정 준수**: 데이터 주권 및 법적 요구사항
2. **지연 시간**: 고객과 가장 가까운 리전
3. **서비스 가용성**: 모든 서비스가 모든 리전에 있지 않음
4. **가격**: 리전마다 가격이 다름

#### 가용 영역 (Availability Zone, AZ)
- 리전 내에 있는 하나 이상의 데이터 센터
- 각 AZ는 독립적인 전원, 네트워크, 연결성을 가짐
- 같은 리전 내 AZ들은 고속 전용 네트워크로 연결
- 서울 리전(ap-northeast-2)은 4개 AZ 보유: a, b, c, d

#### 엣지 로케이션 (Edge Location)
- CloudFront CDN을 위한 캐시 서버
- 현재 400개+ 엣지 로케이션 운영
- 사용자에게 가장 가까운 위치에서 콘텐츠 제공
- Route 53(DNS)도 엣지 로케이션 사용

### 3. AWS 공동 책임 모델 (Shared Responsibility Model)

**⭐ 시험에 매우 자주 출제되는 핵심 개념입니다!**

AWS와 고객은 클라우드 보안에 대한 책임을 공유합니다.

#### AWS의 책임 (클라우드 자체의 보안, Security OF the Cloud)
- 물리적 데이터 센터 보안
- 하드웨어, 소프트웨어, 네트워킹 인프라
- 호스트 운영 체제의 패치
- 관리형 서비스의 기본 인프라

#### 고객의 책임 (클라우드 내부의 보안, Security IN the Cloud)
- 데이터 암호화
- IAM 사용자 및 권한 관리
- 보안 그룹 및 네트워크 ACL 구성
- 게스트 OS 패치 (EC2의 경우)
- 애플리케이션 수준 보안

## ⭐ 핵심 포인트 (시험 출제 빈도 높음)

1. ⭐ **공동 책임 모델**: AWS = 클라우드 자체 보안, 고객 = 클라우드 내 보안
2. ⭐ **가용 영역**: 하나 이상의 데이터 센터로 구성, 각각 독립적 전원/네트워크
3. ⭐ **리전 선택 기준**: 규정 준수 > 지연 시간 > 서비스 가용성 > 가격
4. ⭐ **엣지 로케이션**: 리전/AZ보다 수가 훨씬 많음 (400개+)
5. ⭐ **EC2의 경우**: 고객이 OS 패치 책임 / RDS의 경우: AWS가 DB 엔진 패치 책임

## 💻 실제 예시 - AWS CLI로 리전 정보 확인

```bash
# 사용 가능한 모든 리전 목록 조회
aws ec2 describe-regions --output table

# 특정 리전의 가용 영역 목록 조회
aws ec2 describe-availability-zones \
  --region ap-northeast-2 \
  --query 'AvailabilityZones[*].{Name:ZoneName,State:State}' \
  --output table

# 현재 설정된 리전 확인
aws configure get region
```

**출력 예시:**
```
---------------------------------------------------
|              DescribeAvailabilityZones           |
+---------------------+---------------------------+
|        Name         |          State            |
+---------------------+---------------------------+
| ap-northeast-2a     |        available          |
| ap-northeast-2b     |        available          |
| ap-northeast-2c     |        available          |
| ap-northeast-2d     |        available          |
+---------------------+---------------------------+
```

## 📝 연습 문제

**문제 1.** AWS 공동 책임 모델에서 고객의 책임에 해당하는 것은?

A) 글로벌 인프라 관리  
B) 물리적 데이터 센터 보안  
C) EC2 인스턴스의 운영 체제 패치  
D) 하이퍼바이저 보안  

**정답: C**  
해설: EC2 인스턴스의 OS는 고객이 직접 관리합니다. AWS는 하이퍼바이저 이하의 물리적 인프라를 관리합니다.

---

**문제 2.** AWS 리전을 선택할 때 가장 먼저 고려해야 하는 요소는?

A) 가격  
B) 서비스 가용성  
C) 지연 시간  
D) 규정 준수 요건  

**정답: D**  
해설: 데이터 주권이나 법적 요구사항이 있는 경우 반드시 해당 요건을 충족하는 리전을 선택해야 합니다.

---

**문제 3.** AWS 엣지 로케이션에 대한 설명으로 올바른 것은?

A) 리전보다 수가 적다  
B) 주로 EC2 인스턴스를 호스팅한다  
C) CloudFront CDN의 캐시 서버 역할을 한다  
D) 가용 영역과 동일한 개념이다  

**정답: C**  
해설: 엣지 로케이션은 Amazon CloudFront가 콘텐츠를 캐시하는 서버입니다. 리전(30+)이나 AZ보다 훨씬 많은 400개+ 위치에 존재합니다.

---

**문제 4.** 가용 영역(AZ)에 대한 설명으로 올바르지 않은 것은?

A) 하나 이상의 데이터 센터로 구성된다  
B) 동일 리전 내 AZ들은 고속 전용 네트워크로 연결된다  
C) 각 AZ는 독립적인 전원과 네트워크를 가진다  
D) 전 세계적으로 하나만 존재한다  

**정답: D**  
해설: 가용 영역은 각 리전 내에 여러 개 존재합니다. 서울 리전에만 4개(a, b, c, d)의 AZ가 있습니다.

---

**문제 5.** Amazon RDS를 사용할 때 AWS의 책임 범위에 포함되는 것은?

A) 데이터베이스에 저장된 데이터 암호화  
B) 데이터베이스 사용자 권한 관리  
C) 데이터베이스 엔진 패치 적용  
D) 데이터베이스 백업 보존 기간 설정  

**정답: C**  
해설: RDS는 관리형 서비스이므로 AWS가 데이터베이스 엔진 패치를 담당합니다. 데이터 암호화, 사용자 권한, 백업 설정은 고객의 책임입니다.

## 📌 오늘의 요약

1. 클라우드 컴퓨팅은 온디맨드로 IT 리소스를 제공하며, 초기 비용 절감과 글로벌 확장이 주요 장점이다
2. AWS 글로벌 인프라는 리전 > 가용 영역 > 데이터 센터 구조로 이루어진다
3. 리전 선택 시 규정 준수 > 지연 시간 > 서비스 가용성 > 가격 순으로 고려한다
4. 공동 책임 모델에서 AWS는 인프라 보안, 고객은 데이터와 애플리케이션 보안을 담당한다
5. 엣지 로케이션(400+)은 리전(30+)이나 AZ보다 훨씬 많으며 CloudFront CDN에 사용된다
